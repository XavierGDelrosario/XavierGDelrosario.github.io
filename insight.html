UBC Course Query

The InsightUBC query engine, running entirely in your browser. Add or remove datasets, query any one of them, and overlay two on the same chart to compare.

Edit the InsightUBC query below (the same JSON grammar the real engine accepts) or pick a preset, then run it. A query references one dataset by its id prefix (e.g. sections_avg); set an Overlay to plot a second dataset on the same chart. Results render as a table and a bar chart.

Datasets

Query reference

A query is a JSON object with a WHERE filter and an OPTIONS block, plus an optional TRANSFORMATIONS block for aggregation.

{
  "WHERE":   { ...filter... },
  "OPTIONS": {
    "COLUMNS": ["sections_dept", "sections_avg"],
    "ORDER":   "sections_avg"
  }
}

Datasets

Each dataset has an id, a kind, and a row count — manage them in the Datasets panel: add a built-in sample, or Upload JSON… to add your own. Uploads accept a JSON array of rows, or an object with a result array (the raw UBC format); the fields dept, id, avg, instructor, title, pass, fail, audit, year, uuid are recognized, and the raw keys Subject, Course, Avg, Professor, Title, Pass, Fail, Audit, Year are mapped automatically. The dataset id comes from the file name (no underscore allowed).

A single query may reference only one dataset; referencing two raises an error. To compare datasets, leave the query targeting one dataset and pick a second under Overlay — the same query is re-run against it (its keys re-prefixed automatically) and both series are drawn on one chart.

Fields

Every key is written as <datasetId>_<field> (e.g. sections_avg). The sample datasets share the same fields below — any other key (or a numeric field used where text is expected, and vice-versa) is rejected.

TypeFieldsUsed with
Numericavg pass fail audit yearGT, LT, EQ · AVG, MAX, MIN, SUM
Textdept id instructor title uuidIS (with wildcards)

WHERE — filters

  • Logic: AND / OR take an array of filters; NOT takes a single filter.
  • Numeric comparison: GT, LT, EQ — e.g. { "GT": { "sections_avg": 90 } }.
  • Text match: IS — e.g. { "IS": { "sections_dept": "cpsc" } }. Wildcards: "cpsc*" (starts with), "*cpsc" (ends with), "*cpsc*" (contains). The * is only allowed at the start and/or end.
  • Match everything: an empty "WHERE": {} returns all rows.
{ "AND": [
    { "IS": { "sections_dept": "cpsc" } },
    { "GT": { "sections_avg": 80 } }
] }

OPTIONS

  • COLUMNS (required): a non-empty array of keys to return.
  • ORDER (optional): a single key string, or { "dir": "UP" | "DOWN", "keys": [...] }. Every ORDER key must also appear in COLUMNS.

TRANSFORMATIONS — aggregation (optional)

"TRANSFORMATIONS": {
  "GROUP": ["sections_dept"],
  "APPLY": [
    { "avgGrade":    { "AVG":   "sections_avg" } },
    { "numSections": { "COUNT": "sections_uuid" } }
  ]
}
  • GROUP: one or more keys to group rows by.
  • APPLY tokens: AVG, MAX, MIN, SUM (numeric fields) and COUNT (any field — counts distinct values). AVG/SUM are rounded to two decimals.
  • When transforming, COLUMNS may only contain GROUP keys and the names you define in APPLY.

Queries returning more than 5000 rows raise a ResultTooLargeError. Full project source is on GitHub ↗.
