UBC Course Query

Planning & documentation — a full-stack query engine built in iterative sprints, with the API contract and query grammar designed up front.

How it was planned

  • Tracked work as GitHub Issues using a structured User Story template — each story captured a Role / Goal / Benefit, a Definition of Done, a breakdown of engineering tasks, and a time estimate.
  • Defined the query grammar (an EBNF-style JSON format) and the RESTful API contract before implementation, so the frontend and backend could be built against an agreed interface.
  • Test-first: query behaviours were validated with a Mocha/Chai suite mirroring the spec.

User-story template

  • Summary — Role / Goal / Benefit
  • Definition of done — concrete, testable acceptance criteria
  • Engineering tasks — the steps to build and validate it
  • Time estimate — hours and due date
